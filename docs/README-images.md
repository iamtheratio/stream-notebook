# Screenshots the README expects

The README links these five images. Until they exist, GitHub shows a broken-image
icon in their place — so either add them or delete the matching `![...]` line.

Save them to `docs/img/` with exactly these filenames.

| File | What to capture |
|---|---|
| `overlay-on-stream.png` | The money shot, at the very top of the README. The notebook over real gameplay with 3–4 believable notes in it. This is the one that decides whether anyone reads further — make it look good. |
| `01-start-bat.png` | The unzipped folder in File Explorer, with `start.bat` visible. Highlight or circle it. |
| `02-connect-twitch.png` | The dashboard right after clicking **Connect with Twitch**, showing the 6-character code. **Blur or fake the code** — don't publish a live one. |
| `03-obs-browser-source.png` | The OBS browser-source properties window, URL/width/height filled in and both tickboxes ticked. The single most valuable image here. |
| `04-first-note.png` | The OBS preview with the notebook visible showing one note. |

## Before you commit them

- **Check for anything private** — stream keys, your email, other browser tabs,
  Discord notifications, the real device code.
- **Crop tight.** A full 4K desktop screenshot renders as an unreadable smudge on
  GitHub. Crop to just the window that matters.
- **PNG, and keep them under ~500 KB each.** They live in the repo forever and
  everyone who downloads the ZIP downloads them too.
- Windows: **Win+Shift+S** crops as you capture. For a single window, click it and
  press **Alt+PrtScn**.

## Sizing

GitHub renders README images about 850px wide. Anything wider is scaled down, so
1200–1600px is plenty and keeps the file small.
